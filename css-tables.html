RGB Each parameter (red, green, and blue) defines the intensity of the color with a value between 0 and 255. an extra parameter sets transparency rgb(red,green,blue,opacity) i.e rgb(112,60,143,0.2) link

CSS Selectors

Selector Detail Example Link
Universal Selector * to apply style to all elements *{color:red;} link
Type Selector matches elements by node name. In other words, it selects all elements of the given type within a document. p{color:green;} link
Selector de clase matches elements based on the contents of their class attribute .clase1{color:blue;} link
Selector de ID matches an element based on the value of the element's id attribute. In order for the element to be selected, its id attribute must match exactly the value given in the selector #ID1{color:blue;} link
Selector de atributos matches elements based on the presence or value of a given attribute. td[att]{color:red;} link
matches elements euql to the given attribute. td[att=value]{color:red;} link
matches elements that cointain a given value. td[att ~=value]{color:red;} link
matches elements where the first element is the given attribute. td[att ^=value]{color:red;} link
matches elements where the last element is the given attribute. td[att $=value]{color:red;} link
selects the elementsin the given attribute where some of the data is the attribute td[att *=value]{color:red;} link
pseudo-class Selector pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s). button:hover{...}; button:disabled{...}; link
pseudo-element Selector pseudo-element is a keyword added to a selector that lets you style a specific part of the selected element(s) p::first-line{...} link
Selectors lists CSS selectors define the elements to which a set of CSS rules apply. Sel1,Sel2{...} link

Selectors that utilize a descendant combinator are called descendant selectors. Abuelo nieto{...} link
Child_combinator the child combinator (>) is placed between two CSS selectors. It matches only those elements matched by the second selector that are the direct children of elements matched by the first. parent>child{...} link
sibling_combinator Selector The general sibling combinator (~) separates two selectors and matches all iterations of the second element, that are following the first element (though not necessarily immediately), and are children of the same parent element sibling~sibling2{...} link
Adjacent_sibling_combinator Selector adjacent sibling combinator (+) separates two selectors and matches the second element only if it immediately follows the first element, and both are children of the same parent element sibling+sibling2{...} link
